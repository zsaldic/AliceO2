1 # DPL workflows for the TPC
2 
3 ## TPC reconstruction workflow
4 The TPC reconstruction workflow starts from the TPC digits, the *clusterer* reconstructs clusters in the
5 [ClusterHardware](../../../DataFormats/Detectors/TPC/include/DataFormatsTPC/ClusterHardware.h) format.
6 The clusters are written to RAW pages and passed onto the *decoder* providing the decoded (native) cluster
7 format to the *tracker*.
8 
9 Note: The format of the raw pages is preliminary and does not reflect what is currently implemented in the CRU.
10 
11 The workflow consists of the following DPL processors:
12 
13 * `tpc-digit-reader` -> using tool [o2::framework::RootTreeReader](../../../Framework/Utils/include/Utils/RootTreeReader.h)
14 * `tpc-clusterer` -> interfaces [o2::TPC::HwClusterer](../reconstruction/include/TPCReconstruction/HwClusterer.h)
15 * `tpc-cluster-decoder` -> interfaces [o2::TPC::HardwareClusterDecoder](../reconstruction/include/TPCReconstruction/HardwareClusterDecoder.h)
16 * `tpc-tracker` -> interfaces [o2::TPC::TPCCATracking](../reconstruction/include/TPCReconstruction/TPCCATracking.h)
17 * `tpc-track-writer` -> implements simple writing to ROOT file
18 
19 Depending on the input and output types the default workflow is extended by the following readers and writers:
20 * `tpc-raw-cluster-reader`
21 * `tpc-decoded-cluster-reader`
22 * `tpc-raw-cluster-writer`
23 * `tpc-decoded-cluster-writer`
24 
25 
26 MC labels are passed through the workflow along with the data objects and also written together with the
27 output at the configured stages (see output types).
28 
29 ### Input data
30 The input can be created by running the simulation (`o2sim`) and the digitizer workflow (`digitizer-workflow`).
31 The digitizer workflow produces the file `tpcdigits.root` by default, data is stored in separated branches for
32 all sectors.
33 
34 The workflow can be started starting from digits, raw clusters, or decoded (native) clusters, or directly attached to the
35 `digitizer-workflow`, see comment on inputs types below.
36 
37 ### Quickstart running the reconstruction workflow
38 The workflow is implemented in the `tpc-reco-workflow` executable.
39 
40 Display all options
41 ```
42 tpc-reco-workflow --help
43 ```
44 
45 Important options for the `tpc-digit-reader` as initial publisher
46 ```
47 --infile arg Name of the input file
48 --treename arg (=o2sim) Name of the input tree
49 --digitbranch arg (=TPCDigit) Digit branch
50 --mcbranch arg (=TPCDigitMCTruth) MC label branch
51 --nevents Number of events
52 ```
53 
54 The `tpc-raw-cluster-reader` uses the same options except the branch name configuration
55 --databranch arg (==TPCClusterHw) RAW Cluster branch
56 --mcbranch arg (==TPCClusterHwMCTruth) MC label branch
57 
58 Options for the `tpc-track-writer` process
59 ```
60 --outfile arg (=tpctracks.root) Name of the output file
61 --treename arg (=tpcrec) Name of tree
62 --nevents arg (=-1) Number of events to execute
63 --terminate arg (=process) Terminate the 'process' or 'workflow'
64 --track-branch-name arg (=TPCTracks) configurable branch name for the tracks
65 --trackmc-branch-name arg (=TPCTracksMCTruth) configurable branch name for the MC labels
66 ```
67 
68 Examples:
69 ```
70 tpc-reco-workflow --infile tpcdigits.root --tpc-sectors 0-15 --tracker-options "cont refX=83 bz=-5.0068597793"
71 ```
72 
73 ```
74 tpc-reco-workflow --infile tpcdigits.root --tpc-sectors 0-15 --disable-mc 1 --tracker-options "cont refX=83 bz=-5.0068597793"
75 ```

77 ### Global workflow options:
78 ```
79 --input-type arg (=digits) digitizer, digits, raw, decoded-clusters
80 --output-type arg (=tracks) digits, raw, decoded-clusters, tracks
81 --disable-mc arg (=0) disable sending of MC information
82 --tpc-lanes arg (=1) number of parallel lanes up to the tracker
83 --tpc-sectors arg (=0-35) TPC sector range, e.g. 5-7,8,9
84 ```
85 
86 #### Input Type
87 Input type `digitizer` will create the clusterers with dangling input, this is used
88 to connect the reconstruction workflow directly to the digitizer workflow.
89 
90 All other input types will create a publisher process reading data from branches of
91 a ROOT file. File and branch names are configurable. The MC labels are always read
92 from a parallel branch, the sequence of data and MC objects is assumed to be identical.
93 
94 #### Output Type
95 The output type selects up to which final product the workflow is executed. Multiple outputs
96 are supported in order to write data at intermediate steps, e.g.
97 ```
98 --output-type raw,tracks
99 ```
100 
101 MC label data are stored in corresponding branches per sector. The sequence of MC objects must match
102 the sequence of data objects.
103 
104 #### Parallel processing
105 Parallel processing is controlled by the option `--tpc-lanes n`. The digit reader will fan out to n processing
106 lanes, each with clusterer, converter and decoder. The tracker will fan in from the parallel lanes.
107 For each sector, a dedicated DPL data channel is created. The channels are distributed among the lanes.
108 The default configuration processes sector data belonging together in the same time slice, but in earlier
109 implementations the sector data was distributed among multiple time slices (thus abusing the DPL time
110 slice concept). The tracker spec implements optional buffering of the input data, if the set is not complete
111 within one invocation of the processing function.
112 
113 #### TPC sector selection
114 By default, all TPC sectors are processed by the workflow, option `--tpc-sectors` reduces this to a subset.

118 #### TPC CA tracker
119 The [tracker spec](src/CATrackerSpec.cxx) interfaces the [o2::TPC::TPCCATracking](../reconstruction/include/TPCReconstruction/TPCCATracking.h)
120 worker class which can be initialized using an option string. The processor spec defines the option `--tracker-option`. Currently, the tracker
121 should be run with options:
122 ```
123 --tracker-option "refX=83 cont"
124 ```
125 
126 The most important tracker options are:
127 ```
128 cont activation of continuous mode
129 refX= reference x coordinate, tracker tries to propagate all track to the reference
130 bz= magnetic field
131 ```
